# 星语物语 (StarTale) - 验收检查清单

## 1. 项目结构检查

### 1.1 目录结构
- [ ] 遵循 Feature-First 架构
- [ ] 每个功能模块包含 data/domain/presentation 三层
- [ ] core 目录包含所有基础设施代码
- [ ] shared 目录包含共享组件
- [ ] 无文件超过 1500 行

### 1.2 命名规范
- [ ] 文件名使用 snake_case
- [ ] 类名使用 PascalCase
- [ ] 变量名使用 camelCase
- [ ] 常量使用 camelCase 或 SCREAMING_SNAKE_CASE
- [ ] Provider 名称以 Provider 结尾

### 1.3 导入规范
- [ ] 使用 `@/` 路径别名，禁止 `../../`
- [ ] 导入顺序：dart 核心库 → 第三方库 → 项目内部
- [ ] 无未使用的导入

---

## 2. 代码规范检查

### 2.1 Flutter 规范
- [ ] 所有 Widget 使用 const 构造函数（如适用）
- [ ] StatelessWidget 优先于 StatefulWidget
- [ ] 使用 ConsumerWidget 替代 StatefulWidget + Provider
- [ ] 列表项使用 ListView.builder
- [ ] 避免在 build() 方法中创建对象

### 2.2 Dart 规范
- [ ] 使用 null safety
- [ ] 使用 final 声明不可变变量
- [ ] 使用 late 延迟初始化（必要时）
- [ ] 避免使用 dynamic 类型
- [ ] 使用 ?? 和 ?. 处理空值

### 2.3 状态管理规范
- [ ] 使用 Riverpod 进行状态管理
- [ ] Provider 定义在独立文件中
- [ ] 使用 StateNotifier 管理复杂状态
- [ ] 使用 ProviderScope 覆盖测试

### 2.4 注释规范
- [ ] 文件头包含 JSDoc 注释
- [ ] 公共方法包含文档注释
- [ ] 复杂逻辑包含说明注释
- [ ] 注释使用中文
